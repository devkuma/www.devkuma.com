네트워크란?

네트워크(Network)란 일반적으로 ‘사람이나 물건을 그물로 엮는 것’이라고 할 수 있다. IT 시스템의 세계에서는 ‘컴퓨터간 상호접속을 하는 것’을 의미한다. 그리고 ‘정보가 흐르는 경로’라고 바꿔 불러도 좋을지도 모른다. 통신회로나 케이블 등을 통해서 컴퓨터끼리 접속하는 것으로, 정보의 공유나 처리의 분산, 메신저 교환 등이 가능하다.

네트워트는 단어는 Net + Work의 합성어 인데, 그대로 직역하면 “그물 작업” 라는 뜻이 될것이고, 조금 알기 쉽게 표현하자면, “그물 처럼 서로 얽혀서 협력하여 수행하는 작업” 정도 될것이다. 더 나아가 “컴퓨터들이 서로 연뎔된 상태로 작원을 공유하는 작업"이라고 할수 잇따.

그렇다면, 인터넷은? 전 세계의 네트워크 통신망을 일정한 통신 규약(TCP/IP 프로토콜)을 통해서 통시하는 거대 통대망이다.

네트워크의 구성 요소

네트워크는 ‘컴퓨터’, ‘네트워크 기기’, ‘운송매체’ 등으로 구성된다. 컴퓨터는 PC나 서버 등 어플리케이션(서비스)를 제공하거나, 이용하거나 하는 것이다. 네트워크 기기는 ‘스위치’나 ‘루터’등 어플리케이션의 데이터 운송을 위한 것이다 운송매체와 케이블이나 무선전파같이 컴퓨터와 네트워크 기기를 접속하는 것이다.

네트워크 토폴로지

네트워크 토폴로지(Topology)는 여러가지 네트워크의 물리적인 연결 형태로 네트워크에서 케이블로 연결된 컴퓨터들의 전체적인 모양을 말한다.

  • 물리적인 구조 : 미디어로 연결된 호스트 들의 실제적인 연결 모양
  • 논리적인 구조 : 미디어를 호스트들이 어떤 방식으로 접속하느냐에 따른 네트워크 모양

물리적 토폴로지에는 6가지 형태가 있다.

Bus Topology (버스형)

버스형 토폴로지는 버스라는 공통 배선에 각 노드가 연결된 형태이다. 모든 장비들이 백본이라는 중심 케이블에 연결되어 있다. 이더넷 프로토콜도 이 형태를 사용한다.

  • 장점
    • 노드의 추가 및 삭제가 용이 특정 노드의 장애가 다른 노드에 영향을 주지 않는다.
    • 비용이 저렴하다.
  • 단점
    • 공통 배선의 대역폭을 공유하기 때문에 노드 수가 증가하면 배선의 트래픽이 증가하여 네트워크 성능이 저하된다.
    • 확장하려면 전체를 중지 시켜야 하고, 케이블에 문제가 생기면 찾기가 어렵다.

Star Topology (스타형, 성형)

보는 그대로 별처럼 생긴 토폴로지 이다. 중앙에 각 노드들이 100m 이내에 연결되어 있는 형태이다. 각 노드는 단 하나의 링크의 입출력을 가진다.

  • 장점
    • 장애 발견이 쉽고, 관리가 용이하다.
    • 노드에 문제가 생겨도, 전체 네트워크에 영향이 없다.
    • 중앙에 꽂으면 되니까 확장이 굉장히 용이하다.
  • 단점
    • 주 노드에 장애가 발생하면 전테 네트워크 사용이 불가능하다.
    • 관리는 쉬운데 비해 꼭 중앙을 거쳐서 전송해야 하므로 병목현상이 일어날 가능성이 있다.

Ring Topology (링형)

각 노드의 좌우의 인접한 노드와 연결되어 원형을 이루고 있는 링 형태의 토폴로지이다. 각각 바로 옆 디바이스랑 직접 연결이 되어 있다. 한쪽 방향으로만 데이터 전송이 가능하다. 토큰링도 이 형태다.

  • 장점
    • 단 방향 통신으로 신호 증폭이 가능하여 거리 제약이 적다.
    • 넓은 공간에 설치하기 용이하고 대역폭도 넓게 쓸 수 있다는 장점이 있다.
  • 단점
    • 노드의 추가 삭제가 용이하지 않다.
    • 설치 비용이 비싸다. (공간이 좁으면 내려간다.)

Mesh Topology (망형)

모두 서로서로 다 연결되어 있는 형태의 토폴로지이다. 모든 노드가 서로 일대일로 연결된 그물망 형태로 다수의 노드 쌍이 동시에 통신할 수 있다.

완전 연결형과 부분 연결형이 있다 완전연결형 네트워크는 스위칭 또는 브로드 케스팅이 필요 없으나 노드가 추가 될 때마다 연결 수가 급격하게 증가한다.

n개의 노드에 필요한 연결 수는 n(n-1)/2이다.

  • 장점

    • 특정 노드의 장애가 다른 노드에 영향을 주지 않고 회선장애에 유연한 대처 가능하다.
    • 보안성도 가장 견고하다.

  • 단점

    • 회선 구축 비용이 많이 들며 새로운 노드 추가 시 비용 부담이 발생한다.
    • 설치 및 재조정이 어렵다.

통신 방식

브로드캐스트, 유니캐스트, 멀티캐스트 3가지 모두 통신 방식을 말하는 것이다.

  • 브로드캐스트 (BroadCast) : 하나의 네트워크 전체에 전송 메세지를 전송하는 것이다.
  • 유니캐스트 (UniCast) : 1:1 통신 방식이다.
  • 멀티캐스트 (MultiCast) : 전송할 대상을 골라서 전송하는 것이다.

ISP(Internet Service Provider)
